① 문제 정의

해결하려는 문제 & 대상 사용자

핵심 문제

게임 개발 조직에서 반복적으로 쌓이는 지식 — 메카닉 실험 결과, 가설 검증, 마케팅 성과, 경쟁사 분석 — 이 개인의 기억과 분산된 문서에만 존재하며, 다음 게임 기획 시 활용되지 못하는 지식 단절 문제.

대상 사용자
🎯 주 사용자 — 게임 기획자 (PM)

"이런 메카닉, 전에 해봤나? 결과가 어땠나?"를 바로 알고 싶지만, 그 정보가 시니어 한 명의 머릿속에만 있는 상황

부 사용자
📊 데이터 분석가 📣 마케터 ⚙️ 개발자
현재 업무 방식 — 인터뷰 근거
바울 (Lead PD)

메카닉 실험 결과를 직접 기억에서 꺼내어 팀에 공유 → 바울 1인 의존도 높음

솔 (Junior PD)

과거 실패 사례를 모르고 같은 방향을 시도 → "전에 해봤는데 안 됐어요" 피드백

네이선 (Data Analyst)

지표 데이터와 기획 논의가 연결되지 않아 통합 인사이트 생성 불가

① 문제 정의

주요 불편 사항

A
지식 접근성 문제

과거 게임들의 데이터와 논의 결과가 있지만 다음 게임 기획에 활용이 안 됨. 시니어(바울)에게 직접 물어봐야만 알 수 있는 정보 존재.

B
지식 단절 문제

회의에서 나온 인사이트, 가설, 결과가 체계적으로 기록되지 않아 회의 종료 후 소실됨. 같은 논의가 반복됨.

C
팀 간 데이터 사일로

기획팀의 가설, 데이터팀의 지표, 마케팅팀의 캠페인 성과가 서로 연결되지 않아 통합 인사이트 생성 불가.

D
성공/실패 패턴 미활용

과거 게임들의 성공 패턴과 실패 패턴이 조직 자산으로 축적되지 않아 신규 게임 기획 시 반복 실수 발생.

단순한 도구/프로세스 문제가 아닌, 조직 지식이 개인에게 집중되어 있고 AI로 이를 민주화하지 못한 구조적 문제

⑤ 프로토타이핑 방법

실제 프로토타입 제작 방식 — Cursor Subagents

단순 코딩이 아닌, Cursor Subagents를 활용한 AI 협업 개발 방법론으로 구현했습니다. 기획자 · 디자이너 · 개발자 3개 Agent가 순차적으로 산출물을 생성하며, 각 Agent는 이전 Agent의 산출물을 참조 문서로 활용합니다.

Step 1
🧠 Product Manager Agent
Feature_Spec.md · Db_Schema.md · Api_Spec.md 작성
Step 2
🎨 Product Designer Agent
6개 HTML/CSS 목업 + Ui_Spec.md 작성
Step 3
⚙️ Fullstack Developer Agent
Django 풀스택 구현 (models · views · templates · services)
프롬프트 위치
.cursor/agents/product-manager-agent.md .cursor/agents/product-designer-agent.md .cursor/agents/fullstack-developer-agent.md

Product Manager Agent

Product Designer Agent
역할: SuperSuit AI의 제품 디자이너 참조: PM Agent 산출물 산출물: - Ui_Spec.md (디자인 시스템 + 화면별 UI) - mockups/01~06.html (6개 HTML/CSS 목업) 기술 요구사항: - Tailwind CSS CDN - vis.js CDN (그래프) - 독립 실행 가능한 HTML - 다크 사이드바 + 라이트 콘텐츠 - 반응형 (최소 1280px)
Fullstack Developer Agent
역할: SuperSuit AI의 풀스택 개발자 참조: 모든 docs/ + mockups/ 구현 항목: - models.py (11개 노드) - clova_service.py (STT) - gpt_service.py (엔티티 추출 + LLM) - ontology_service.py (노드/엣지 저장) - 6개 Django 템플릿 - fixtures/sample_data.json 기술: Django 5.2 + pgvector Docker + gpt-5.4-mini
