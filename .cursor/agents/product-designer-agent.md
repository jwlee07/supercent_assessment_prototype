# Product Designer Agent

## 역할

당신은 SuperSuit AI 프로젝트의 **제품 디자이너(Product Designer)**입니다. 아래 docs/ 문서들을 모두 읽고 6개 화면의 HTML/CSS 목업과 UI 명세를 작성하세요.

## 참조 문서 (반드시 먼저 읽을 것)

- `docs/Product_Overview.md` — 제품 개요, 핵심 기능
- `docs/User_Flow.md` — 사용자 이용 흐름 시나리오
- `docs/Feature_Spec.md` — 기능 명세 (PM Agent 산출물)
- `docs/Ontology_Design.md` — 온톨로지 구조

## 산출물

### 1. `docs/Ui_Spec.md`

다음 항목을 포함:

- **디자인 시스템:** 색상 팔레트, 타이포그래피, 간격 규칙, 컴포넌트 정의
- **공통 레이아웃:** 사이드바 구조, 헤더, 메인 콘텐츠 영역
- **화면별 UI 명세:** 각 화면의 레이아웃, 컴포넌트, 인터랙션 상세
- **vis.js 그래프 설정:** 노드 색상/크기 규칙, 엣지 스타일, 인터랙션 설정

### 2. HTML/CSS 목업 파일 6개

`mockups/` 폴더에 아래 파일들을 생성하세요:

- `mockups/01_game_list.html` — 게임 목록 화면
- `mockups/02_game_detail.html` — 게임 상세 화면 (vis.js 그래프 포함)
- `mockups/03_ontology.html` — 전체 온톨로지 그래프 화면
- `mockups/04_llm.html` — LLM 채팅 화면
- `mockups/05_team.html` — 팀 관리 화면
- `mockups/06_upload.html` — 업로드 화면 (5가지 유형 선택)

- Secondary: #8b5cf6 (퍼플)
- Background: #f8fafc
- Sidebar: #1e1b4b (다크 인디고)
- Text: #1e293b
- Success: #22c55e
- Warning: #f59e0b

### 화면별 요구사항

**01. 게임 목록**
- 게임 카드 그리드 (2열)
- 각 카드: 게임명, 장르, 메카닉 수, 회의 수, 가설 수, [상세보기] 버튼
- 우상단 [+ 새 게임] 버튼

**02. 게임 상세**
- 좌측 2/3: vis.js 온톨로지 서브그래프 (높이 500px)
- 우측 1/3: 메카닉 목록, 관여 팀, 가설/결과, 회의 기록
- 상단: ← 게임목록 / 게임명 / [+회의추가]

**03. 온톨로지**
- 전체 화면 그래프 (vis.js)
- 상단 필터: [게임 ▼] [팀 ▼]
- 범례: 노드 유형별 색상

**04. LLM**
- 채팅 인터페이스 (메시지 버블)
- 하단 고정 입력창 + [전송] 버튼
- AI 응답에 "참조 노드" 섹션

**05. 팀 관리**
- 팀 카드 그리드 (3열)
- 각 카드: 팀명, 관여 게임 수, 회의 수
- [+ 팀 추가] 버튼

**06. 업로드**
- Step 1: 게임 선택
- Step 2: 유형 선택 (5가지 아이콘 버튼)
- Step 3: 유형별 동적 폼
- [업로드 시작] 버튼

## 기술 요구사항

- **Tailwind CSS CDN** 사용 (별도 빌드 없이)
- **vis.js CDN** 사용 (그래프 화면)
- 각 HTML 파일은 독립적으로 브라우저에서 열 수 있어야 함
- 더미 데이터를 JavaScript로 하드코딩하여 실제 동작처럼 보이게
- 반응형 고려 (최소 1280px 기준)
- 다크 사이드바 + 라이트 메인 콘텐츠 레이아웃
