# 문제 정의 (Problem Definition)

## 해결하려는 문제

게임 개발 조직에서 반복적으로 쌓이는 지식(메카닉 실험 결과, 가설 검증, 마케팅 성과, 경쟁사 분석)이 개인의 기억과 분산된 문서에만 존재하며, 다음 게임 기획 시 활용되지 못하는 **지식 단절 문제**.

## 대상 사용자

**게임 기획자 (PM)**

새 게임 아이디어를 낼 때 "이런 메카닉, 전에 해봤나? 결과가 어땠나?"를 바로 알고 싶지만, 그 정보가 시니어 한 명의 머릿속에만 있거나 찾기 어려운 문서 속에 묻혀 있는 상황에 처한 사람.

## 현재 업무 방식 (인터뷰 근거)

### 바울 (Lead PD) 인터뷰에서
- 신규 게임 기획 시 과거 게임들의 메카닉 실험 결과를 직접 기억에서 꺼내어 팀에 공유
- 팀원들이 "이거 전에 해봤나요?"라고 물으면 바울이 답변하는 구조 → 바울 의존도 높음
- 회의에서 나온 가설과 결과가 체계적으로 기록되지 않아 반복 논의 발생

### 솔 (Game Designer, 현업 10년) 인터뷰에서
- "코어한 플레이를 준비했다 처참하게 실패하고 잽싸게 롤백" — 실패들이 중요한 자산이라 말하지만, 팀 밖으로 전달되려면 구두·문서 커뮤니케이션을 모두 동원해야 해서 공유되지 않는 실패가 생김
- 회의 내용이 개인 노트에만 남아 팀 공유가 안 됨

### 네이선 (Game Designer, Dinosaur Universe 담당) 인터뷰에서
- 라이브옵스·FTUE 개선으로 의미있는 성과를 냈지만, 유저 행동 패턴·리텐션·수익 지표 인사이트가 개인 역량에 머물고 조직에 누적되지 않음
- 데이터 분석 역량이 개인 차원에 그치고 다음 기획자에게 넘어가는 구조가 없음

## 주요 불편 사항

### A. 지식 접근성 문제
과거 게임들의 데이터와 논의 결과가 있지만 다음 게임 기획에 활용이 안 됨. 시니어(바울)에게 직접 물어봐야만 알 수 있는 정보가 존재.

### B. 지식 단절 문제
회의에서 나온 인사이트, 가설, 결과가 체계적으로 기록되지 않아 회의 종료 후 부분 소실되고, 관점에 따라 다르게 이해된다.

### C. 팀 간 데이터 사일로
기획팀의 가설, 데이터팀의 지표, 마케팅팀의 캠페인 성과 등이 각자의 영역에 머물고, 전사적으로 통합 해석되는 표준화된 구조가 없다.

### D. 성공/실패 패턴 미활용
과거 게임들의 성공 패턴과 실패 패턴이 조직 자산으로 축적되지 않아 신규 게임 기획 시 반복 실수 발생.

## 문제의 본질

단순한 도구/프로세스 문제가 아닌, **조직 지식이 개인에게 집중되어 있고 AI로 이를 민주화하지 못한 구조적 문제**.

SuperSuit AI는 이를 해결하기 위해 모든 팀의 데이터를 하나의 온톨로지로 연결하고, 누구나 자연어로 과거 지식에 접근할 수 있게 만든다.
