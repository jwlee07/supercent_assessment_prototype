# 사용자 이용 흐름 (User Flow)

## 시나리오 A: 데이터 업로드 → 온톨로지 축적

### 트리거
기획팀이 Snake Clash! 관련 회의를 마쳤거나, 마케팅팀이 캠페인 결과 CSV를 받았거나, 데이터팀이 지표 리포트를 완성한 상황.

### 흐름

```
1. 사이드바에서 [+ 업로드] 클릭
        ↓
2. 게임 선택
   - Snake Clash! / Dinosaur Universe / + 새 게임 추가
        ↓
3. 데이터 유형 선택 (5가지 중 1개)
   ┌─────────────────────────────────────────┐
   │ 🎙️ 회의 음성  │ 📊 게임 지표 │ 📣 마케팅 │
   │ 🔍 경쟁 리서치 │ 🧪 A/B 테스트              │
   └─────────────────────────────────────────┘
        ↓
4. 유형별 입력 폼 작성
   [회의 음성] 음성 파일 드래그 + 참여 팀 선택 + 날짜
   [게임 지표] CSV 업로드 + 지표 기간 + 지표 유형
   [마케팅]   CSV 업로드 또는 직접 입력 + 캠페인명 + 채널
   [경쟁리서치] PDF/TXT 업로드 또는 텍스트 입력 + 경쟁 게임명
   [A/B 테스트] CSV 업로드 + 가설 연결 + 대조/실험군 결과
        ↓
5. [업로드 시작] 클릭
        ↓
6. 서버 처리 (백그라운드)
   [회의 음성] CLOVA STT → 텍스트 변환
   [모든 유형] gpt-5.4-mini 엔티티 추출 → JSON 노드/엣지
   → PostgreSQL 저장 + text-embedding-3-small 임베딩 생성
        ↓
7. 완료 알림 + 온톨로지 그래프 화면으로 이동
   → 새로 추가된 노드/엣지가 그래프에 표시됨
```

### 결과
회의/데이터가 쌓일수록 온톨로지 그래프가 더 촘촘해지고, LLM 답변의 정확도가 높아진다.

## 시나리오 B: 자연어 질문 → LLM 인사이트 응답

### 트리거
기획자(PM)가 새 게임 아이디어를 구상 중이거나, 특정 메카닉의 과거 성과가 궁금한 상황.

### 흐름

```
1. 사이드바에서 [🤖 LLM] 클릭
        ↓
2. 질문 입력란에 자연어로 질문 입력
   예: "Snake Clash! 메카닉 성공 사례 알려줘"
   예: "D1이 40% 이상인 게임들의 공통점이 뭐야?"
   예: "마케팅 CPI가 가장 낮았던 게임과 그 이유"
        ↓
3. [전송] 클릭
        ↓
4. 서버 처리
   → 질문 텍스트 → text-embedding-3-small → 벡터 변환
   → pgvector에서 코사인 유사도 검색 (상위 10개 노드)
   → 연결된 노드 JOIN (게임, 메카닉, 가설, 결과, 지표, 팀)
   → gpt-5.4-mini에 컨텍스트 + 질문 전달
        ↓
5. LLM 응답 표시
   - 관련 게임/메카닉/수치를 근거로 한 자연어 인사이트
   - 인용된 온톨로지 노드 목록 표시 (출처 투명성)
        ↓
6. (선택) 연관 게임/노드 클릭 → 게임 상세 화면으로 이동
```

### 예시 질답

**질문:** "Snake Clash! 메카닉 성공 사례 알려줘"

**응답:**
> Snake Clash!에서 뱀 이동 루프 메카닉을 단순화한 결과 D1 42%를 달성했습니다 (업계 평균 대비 +7%). 기획팀의 "조작 단순화 → D1 상승" 가설이 검증되었으며, 배틀로얄 요소와 결합 시 DAU +35% 효과가 확인되었습니다. 다음 게임에서 Snake Clash! 메카닉 적용 시 조작 복잡도를 최소화하는 것이 핵심입니다.
>
> 참조 노드: Snake Clash!, 뱀 이동 루프, 조작 단순화 가설, D1 42% 결과

## 시나리오 C: 게임 상세 — 온톨로지 탐색

### 흐름

```
1. [게임 목록] 화면에서 게임 카드 클릭
        ↓
2. 게임 상세 화면 진입
   - 좌측: 해당 게임의 서브 온톨로지 그래프 (vis.js)
   - 우측: 메카닉 목록, 관여 팀, 가설/결과, 회의 기록
        ↓
3. 그래프에서 노드 클릭 → 해당 노드 상세 정보 사이드패널 표시
   예: 메카닉 노드 클릭 → 관련 가설, 결과, A/B 테스트 내역 표시
        ↓
4. [+ 회의 추가] 버튼 → 업로드 화면으로 이동 (해당 게임 선택 유지)
```

## 시나리오 D: 팀 관리 — 팀별 필터링

### 흐름

```
1. [👥 팀 관리] 화면 진입
        ↓
2. 팀 카드 목록 확인
   - 각 팀별 관여 게임 수, 회의 수 요약
        ↓
3. 팀 카드 클릭 → 해당 팀이 관여한 게임/회의/메카닉만 필터링
        ↓
4. 온톨로지 화면에서도 팀 필터 적용 가능
   → 특정 팀이 관여한 노드만 하이라이트
```
