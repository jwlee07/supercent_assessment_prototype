# 기능 명세 (Feature Spec)

## 1. 화면 목록

| 화면 | URL | 설명 |
|---|---|---|
| 게임 목록 | `/` | 전체 게임 카드 그리드 |
| 게임 상세 | `/games/<id>/` | 게임별 온톨로지 서브그래프 + 상세 정보 |
| 온톨로지 | `/ontology/` | 전체 온톨로지 그래프 |
| LLM 질문 | `/ask/` | 자연어 질문 → 온톨로지 기반 응답 |
| 팀 관리 | `/teams/` | 팀별 관여 게임/회의 관리 |
| 업로드 | `/upload/` | 5가지 유형 데이터 업로드 |

---

## 2. 화면별 기능 상세

### 2.1 게임 목록 (`/`)

**UI 요소:**
- 상단 헤더: "게임 목록" 제목 + [+ 새 게임] 버튼
- 게임 카드 그리드 (2열)
- 각 게임 카드:
  - 게임명 (대제목)
  - 장르 배지
  - 메카닉 수 / 회의 수 / 가설 수 통계
  - [상세보기] 버튼

**사용자 액션:**
- [+ 새 게임] → 게임명/장르 입력 모달 → 저장 후 게임 상세 이동
- [상세보기] → 게임 상세 화면 이동

**시스템 동작:**
- 전체 Game 목록 조회 + 각 게임별 관련 노드 수 집계
- 게임 없을 시 빈 상태 안내 메시지 표시

---

### 2.2 게임 상세 (`/games/<id>/`)

**UI 요소:**
- 상단 네비게이션: [← 게임목록] / 게임명 / [+데이터 추가] 버튼
- 좌측 패널 (2/3): vis.js 서브 온톨로지 그래프
  - 노드 클릭 시 우측 패널에 상세 정보 표시
  - 드래그/줌 지원
- 우측 패널 (1/3):
  - 메카닉 목록 (클릭 가능)
  - 관여 팀 배지
  - 가설/결과 목록 (상태: ✅검증됨 / ❌실패 / ⏳진행중)
  - 회의 기록 목록 (날짜, 제목)

**사용자 액션:**
- 그래프 노드 클릭 → 우측 패널에 해당 노드 상세 정보
- [+데이터 추가] → 업로드 화면으로 이동 (해당 게임 선택 유지)
- 회의 기록 클릭 → 회의 내용 모달

**시스템 동작:**
- 해당 게임 관련 노드/엣지 전체 조회
- vis.js용 JSON 생성 (노드 유형별 색상 적용)

---

### 2.3 온톨로지 전체 (`/ontology/`)

**UI 요소:**
- 상단 필터바: [게임 ▼ 전체] [팀 ▼ 전체]
- 전체 화면 vis.js 그래프
- 우하단 범례: 노드 유형별 색상 표시
- 노드 클릭 시 팝업으로 노드 정보 표시

**사용자 액션:**
- 게임 필터 선택 → 해당 게임 관련 노드만 표시
- 팀 필터 선택 → 해당 팀이 관여한 노드만 하이라이트
- 노드 클릭 → 해당 노드 상세 팝업

**시스템 동작:**
- `/ontology/data/` API로 전체 그래프 JSON 동적 로드
- 필터 변경 시 API 재호출

---

### 2.4 LLM 질문 (`/ask/`)

**UI 요소:**
- 상단 헤더: "SuperSuit AI에게 질문"
- 채팅 메시지 영역 (스크롤)
  - 사용자 메시지: 우측 정렬, 인디고 배경
  - AI 응답: 좌측 정렬, 흰색 배경
  - AI 응답 하단: "참조 노드" 섹션 (클릭 시 해당 화면 이동)
- 하단 고정 입력창 + [전송] 버튼
- 예시 질문 버튼 (첫 화면):
  - "뱀 메카닉 성공 사례 알려줘"
  - "D1이 높은 게임들의 공통점은?"
  - "마케팅 CPI가 가장 낮은 게임은?"

**사용자 액션:**
- 질문 입력 후 [전송] 또는 Enter → 응답 표시
- 참조 노드 클릭 → 해당 게임 상세 화면 이동

**시스템 동작:**
1. 질문 텍스트 → text-embedding-3-small → 벡터 변환
2. pgvector `<->` 연산자로 상위 10개 유사 노드 검색
3. 연결 노드 JOIN 조회 (게임, 메카닉, 가설, 결과, 지표)
4. gpt-5.4-mini에 컨텍스트 + 질문 전달
5. 자연어 응답 + 참조 노드 목록 반환

---

### 2.5 팀 관리 (`/teams/`)

**UI 요소:**
- 상단 헤더: "팀 관리" + [+ 팀 추가] 버튼
- 팀 카드 그리드 (3열)
  - 팀명, 관여 게임 수, 회의 수
  - [상세보기] 버튼

**사용자 액션:**
- [+ 팀 추가] → 팀명 입력 모달 → 저장
- [상세보기] → 해당 팀 관여 게임 목록 + 회의 목록 표시

**시스템 동작:**
- Team 목록 + 각 팀별 game/meeting 수 집계
- 팀별 필터 → 온톨로지 화면에서도 연동

---

### 2.6 업로드 (`/upload/`)

**UI 요소:**
- Step 1: 게임 선택 (라디오 버튼)
- Step 2: 유형 선택 (5가지 아이콘 카드)
- Step 3: 유형별 동적 폼
- [업로드 시작] 버튼
- 처리 중 로딩 스피너 + 진행 상태 메시지

**유형별 폼:**

| 유형 | 폼 요소 |
|---|---|
| 회의 음성 | 파일 업로드 + 마이크 녹음 버튼 + 참여 팀 다중 선택 + 날짜 |
| 게임 지표 | CSV/JSON 업로드 + 지표 기간 + 지표 유형 다중 선택 |
| 마케팅 성과 | CSV 업로드 또는 직접 입력 폼 (캠페인명, 채널, CPI, ROAS, CTR) |
| 경쟁 리서치 | PDF/TXT 업로드 또는 텍스트 에리어 + 경쟁 게임명 |
| A/B 테스트 | CSV 업로드 또는 직접 입력 폼 + 연결 가설 드롭다운 |

**사용자 액션:**
- 파일 드래그앤드롭 또는 클릭 업로드
- [업로드 시작] → 처리 진행 → 완료 후 온톨로지 화면 이동

**시스템 동작:**
- 유형별 처리 파이프라인 실행 (Upload_Types.md 참조)
- 처리 완료 후 성공 알림 + 온톨로지 화면으로 리디렉션

## 3. 공통 기능

### 사이드바 네비게이션
- 항상 표시 (고정)
- 현재 활성 메뉴 하이라이트
- 메뉴: 게임 목록, 온톨로지, LLM, 팀 관리
- 하단: [+ 업로드] 버튼

### 오류 처리
- API 호출 실패 시 사용자 친화적 에러 메시지
- 파일 형식 불일치 시 즉시 안내
- CLOVA/GPT API 오류 시 재시도 안내
